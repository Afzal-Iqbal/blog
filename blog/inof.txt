Non react website
A user types a URL in the browser and hits enter.

The browser sends a request to the server for that specific address.

The server responds by sending back a full HTML page.

When the user clicks a link (e.g., to a contact page), the browser sends a new request to the server.

The server returns a new HTML page for that route.

This cycle repeats for every navigation, resulting in multiple full-page reloads

Client-Side Routing




React Router DOM enables client-side routing, meaning navigation is handled entirely in the browser.
Instead of sending a new request to the server for each page, React intercepts the navigation and injects the correct component into the DOM.
